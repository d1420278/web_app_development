# 產品需求文件 (PRD) - 校園惜食匹配系統 (Smart Campus Food Matching)

## 1. 專案概述

### 1.1 背景與動機
隨著環境永續意識的抬頭，食物浪費已成為全球關注的議題。在校園環境中，尤其在每日供餐結束後，常會有剩餘卻依然完好、新鮮的餐點因為過了銷售時間而被迫丟棄。本專案希望能透過數位化平台，將校園內的剩食資訊即時公開，讓有需求的學生能以優惠價格或免費取得這些食物，達成減少浪費、資源最大化利用的雙贏局面。

### 1.2 目標用戶
- **校園餐廳業者（供應端）**：需要有效減少廚餘處理成本、減少食物浪費，甚至能回收些微成本的商家。
- **在校學生（需求端）**：希望能以較低價格取得餐點，或對環境永續有認同感並願意採取行動的學生。
- **行政單位（管理與推廣端）**：負責校園永續發展指標的推動，需統整全校減碳與惜食成效。

### 1.3 核心價值主張
- **環保減碳**：減少校園食物浪費，具體降低剩食產生的碳足跡。
- **經濟實惠**：學生能用更便宜的價格購得餐點，降低生活負擔。
- **即時便利**：平台即時顯示剩食資訊與地圖，讓資訊傳遞零時差。

---

## 2. 功能需求

### 2.1 主要功能與使用者故事 (User Stories)

1. **剩食發佈系統**
   - **作為** 校園餐廳業者，**我希望** 能快速上傳剩餘餐點的照片、份量及預計下架時間，**以便** 即時通知學生前來領取或購買。
2. **即時地圖顯示**
   - **作為** 在校學生，**我希望** 能透過地圖查看校園內各供餐點的剩食分佈與數量，**以便** 快速找到離我最近且還有剩食的餐廳。
3. **預約取餐功能**
   - **作為** 在校學生，**我希望** 能在線上提前保留一份餐點，**以便** 確保我抵達現場時不會撲空。
4. **剩食量統計看板**
   - **作為** 行政單位或平台使用者，**我希望** 能看到系統統計本週/本月減少的剩食重量與換算的碳排放減少量，**以便** 了解並展示惜食行動對環境永續的實際影響。
5. **學生身份驗證**
   - **作為** 平台管理者，**我希望** 系統能夠整合校園專屬帳號 (如 SSO 或學校信箱驗證) 登入，**以便** 確保平台上的資源專屬於校內師生，避免外部人員濫用。

## 3. 非功能需求

### 3.1 技術限制
- **後端框架**：使用 Python 的 Flask 框架進行開發。
- **前端渲染**：使用 Jinja2 模板引擎處理動態網頁。
- **資料庫**：使用 SQLite 輕量級關聯式資料庫，便於開發與部署。

### 3.2 效能與安全考量
- **響應式設計 (RWD)**：介面需針對手機瀏覽器進行優化，因為多數學生會在校園走動時透過手機查看。
- **安全性考量**：
  - 防止重複預約與機器人惡意佔名額，需加入基本防禦機制 (如防護 CSRF、限制預約次數)。
  - 學生的登入憑證與個人資料需安全儲存 (如密碼 Hash 加密)，避免資料外洩。
  - 需驗證上傳圖片的格式與大小，防止伺服器空間耗盡。

## 4. MVP 範圍 (Minimum Viable Product)

### Must Have (必須具備)
- 【餐廳端】剩食發佈 (新增、修改、刪除、設定下架時間)。
- 【學生端】學生基本登入與註冊註冊 (含校內信箱驗證)。
- 【學生端】查看剩食列表與剩食詳情。
- 【學生端】線上預約餐點功能。
- 【餐廳端】查看並核銷學生的預約訂單。

### Should Have (應該具備)
- 【學生端/行政端】剩食量統計看板 (系統簡單統計累積的搶救餐點數量，換算碳排放減少量)。
- 【學生端】即時地圖顯示，在校園地圖上標示有哪些餐廳供應中。

### Nice to Have (有了更好)
- 取餐提醒通知 (透過 Email 或是推播在下架前 15 分鐘提醒取餐)。
- 積分與成就系統，鼓勵學生或餐廳積極參與並頒發數位徽章。
